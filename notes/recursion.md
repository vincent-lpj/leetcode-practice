# Recursion Notes

## 递归基础

### 概念

递归是函数**调用自身**来解决问题的编程技巧。其核心思想是：将一个大问题分解为若干个结构相同、规模更小的子问题，直到子问题足够简单可以直接求解。

### 递归三要素

| 要素         | 说明                                           |
| ------------ | ---------------------------------------------- |
| 终止条件     | 最小子问题的解，不再继续递归，直接返回结果     |
| 当前层的逻辑 | 在这一层做什么（选择、计算、记录）             |
| 缩小问题规模 | 调用自身，传入规模更小的参数（比如 index + 1） |

缺少任意一项都会导致错误：没有终止条件会无限递归，没有缩小规模同样会导致无限递归，最终栈溢出。
终止条件必须放在递归调用之前，否则即使触发终止条件也会不断递归直至溢出

### Python 递归框架

```python
def dfs(参数):
    # 1. 终止条件
    if 到达边界:
        记录或返回结果
        return

    # 2. 当前层逻辑 + 缩小规模
    for 每个可能的选择:
        dfs(下一层参数)
```

## 回溯 (Backtracking)

### 概念

回溯是递归在"枚举所有可能解"场景下的具体应用。它像一棵**决策树**：从根节点出发，每一层做一次选择，走到叶子节点时得到一个完整的解。

与普通递归的区别：回溯通常需要在递归返回后**撤销选择**（restore state），以便尝试其他分支。

> **剪枝 (Pruning)**：在递归中提前检查当前路径是否已经不合法，如果是，直接 `return` 而不继续深入。剪枝能大幅减少无效搜索，是回溯的关键优化。

### 通用框架

```python
result = []

def dfs(当前状态):
    # 终止条件：得到一个完整的解
    if 到达终点:
        result.append(当前路径)
        return

    for 每个可能的选择:
        # 剪枝：提前排除不合法的选择
        if 不合法:
            continue  # 或 return（取决于剪枝类型）

        # 做选择
        更新状态

        # 递归
        dfs(下一层状态)

        # 撤销选择（如果状态是可变对象，比如 list）
        还原状态

dfs(初始状态)
return result
```

> **注意**：如果状态是**不可变对象**（如 Python 的 `str`、`int`），每次递归传入新值，天然不需要撤销。如果状态是**可变对象**（如 `list`），需要在递归返回后手动撤销。

## 常用技巧总结

### 按层递推

用于"每层做一个独立选择，直到所有层都选完"的问题。

**代表题目**：17 - Letter Combinations of a Phone Number

**核心模式**：用 `index` 表示当前处理到第几层（第几个数字），每层遍历该位置所有可能的选项，递归到下一层。

```python
def dfs(index: int, path: str):
    # 终止：所有位置都已选完
    if index == len(digits):
        result.append(path)
        return

    # 枚举当前位置的所有字母
    for ch in phone[digits[index]]:
        dfs(index + 1, path + ch)  # str 不可变，天然撤销

dfs(0, "")
```

决策树示意（digits = "23"）：

```
              ""
         /    |    \
        a     b     c
      / | \ / | \ / | \
     d  e f d e f d e f
```

每条从根到叶的路径，就是一个合法的字母组合。

### 约束剪枝

用于"每步有多种选择，但需要实时检查合法性"的问题。

**代表题目**：22 - Generate Parentheses

**核心模式**：在每个递归分支入口处，先判断当前状态是否合法（剪枝），再继续递归；当状态完整时收集结果。

```python
def dfs(left_count: int, right_count: int, path: str):
    # 剪枝：括号数量超限，或右括号多于左括号（不合法前缀）
    if left_count > n or right_count > n or right_count > left_count:
        return

    # 终止：左右括号都用完，得到合法组合
    if left_count == n and right_count == n:
        result.append(path)
        return

    # 选择 1：加左括号
    dfs(left_count + 1, right_count, path + "(")

    # 选择 2：加右括号
    dfs(left_count, right_count + 1, path + ")")

dfs(0, 0, "")
```

剪枝去掉的分支（以 n=2 为例）：

```
")" → 右括号先于左括号，立刻剪枝
"(((" → 左括号超过 n，立刻剪枝
"())" → 右括号数量超过左括号，立刻剪枝
```

### 总结

| 技巧     | 核心机制                     | 状态是否需要撤销        | 典型题目 |
| -------- | ---------------------------- | ----------------------- | -------- |
| 按层递推 | index 逐层推进，枚举每层选项 | 否（str/int 不可变）    | 17       |
| 约束剪枝 | 实时检查合法性，提前截断分支 | 否（str/int 不可变）    | 22       |
| 显式撤销 | list 等可变对象需手动 pop    | 是（path.pop() 等操作） | 46, 78   |
