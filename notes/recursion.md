# Recursion Notes

## 递归基础

### 概念

递归是函数**调用自身**来解决问题的编程技巧。其核心思想是：将一个大问题分解为若干个结构相同、规模更小的子问题，直到子问题足够简单可以直接求解。

### 递归三要素

| 要素         | 说明                                           |
| ------------ | ---------------------------------------------- |
| 终止条件     | 最小子问题的解，不再继续递归，直接返回结果     |
| 当前层的逻辑 | 在这一层做什么（选择、计算、记录）             |
| 缩小问题规模 | 调用自身，传入规模更小的参数（比如 index + 1） |

缺少任意一项都会导致错误：没有终止条件会无限递归，没有缩小规模同样会导致无限递归，最终栈溢出。
终止条件必须放在递归调用之前，否则即使触发终止条件也会不断递归直至溢出

### Python 递归框架

递归分两种模式，核心区别在于**每层有几条路**：

---

#### 模式一：普通递归（线性分解）

每层只有**一条路**往下走，问题规模逐步缩小，最终触底返回。

```python
def recurse(参数):
    if 到达边界:          # 终止条件
        return 最小结果

    return 当前逻辑 + recurse(缩小后的参数)
```

调用结构是一条直链，**向下递进，向上归并**：

```
recurse(n)
  └── recurse(n-1)
        └── recurse(n-2)
              └── ...
                    └── recurse(0)  ← 触底
                    ↑ 返回并逐层计算
```

典型例子：factorial、fibonacci、reverse string、linked list traversal

---

#### 模式二：回溯递归（枚举所有分支）

每层有**多条路**可以选择，需要穷举所有可能，形成一棵决策树。

```python
def dfs(当前状态):
    if 到达终点:          # 终止条件
        记录结果
        return

    for 每个可能的选择:   # 枚举所有分支
        更新状态
        dfs(下一层状态)
        还原状态          # 撤销选择（可变对象才需要）
```

调用结构是一棵树，**每层展开多个子节点**：

```
dfs("")
├── dfs("a")
│     ├── dfs("ad")  ← 叶子，记录结果
│     ├── dfs("ae")  ← 叶子
│     └── dfs("af")  ← 叶子
├── dfs("b")
│     ├── dfs("bd")
│     └── ...
└── dfs("c")
      └── ...
```

典型例子：#17 Letter Combinations、#46 Permutations、#22 Generate Parentheses

---

> **判断用哪种的简单方法**：
> 问题是"求一个答案"用普通递归；问题是"求所有答案"用回溯递归。

---

### 模式二的两种常见写法

模式二（回溯）在展开分支时有两种写法，选哪种取决于**每层的选项是否固定**：

---

#### 写法 A：for 循环展开（每层选项来自集合）

每层从一个动态集合里选一项，用循环统一处理所有分支。

```python
# #17 Letter Combinations — 每层从对应按键的字母集合里选一个
def dfs(index: int, path: str):
    if index == len(digits):
        result.append(path)
        return

    for ch in phone[digits[index]]:   # ← 循环展开所有字母
        dfs(index + 1, path + ch)
```

决策树每层的宽度由集合大小决定，可以是 3 个也可以是 4 个：

```
         ""
    /    |    \
   a     b     c        ← phone["2"] = "abc"
 / | \ / | \ / | \
d  e f d e f d e f      ← phone["3"] = "def"
```

**适用场景**：每层选项来自列表/字符串/数组，且分支数量不固定。

---

#### 写法 B：固定分支显式展开（每层选项固定且少）

每层只有固定几个选择，直接写出每条 `dfs` 调用，不用循环。

```python
# #78 Subsets — 每层只有两个选择：选 nums[index] 或不选
def dfs(index: int):
    if index == len(nums):
        result.append(path[:])   # 每个叶子都是一个合法子集
        return

    # 选择 1：不选 nums[index]
    dfs(index + 1)

    # 选择 2：选 nums[index]
    path.append(nums[index])
    dfs(index + 1)
    path.pop()                   # 回溯，撤销选择
```

决策树每层宽度固定为 2（以 nums = [1,2,3] 为例）：

```
dfs(0)
├── 不选 1 → dfs(1)
│     ├── 不选 2 → dfs(2)
│     │     ├── 不选 3 → []
│     │     └── 选 3   → [3]
│     └── 选 2  → dfs(2)
│           ├── 不选 3 → [2]
│           └── 选 3   → [2,3]
└── 选 1  → dfs(1)
      ├── 不选 2 → dfs(2) → [1] / [1,3]
      └── 选 2  → dfs(2) → [1,2] / [1,2,3]
```

**适用场景**：每层选项固定（二选一、三选一），如元素选/不选、树的左右子节点、括号加/不加。

---

#### 对比

|            | 写法 A：for 循环             | 写法 B：显式展开     |
| ---------- | ---------------------------- | -------------------- |
| 每层分支数 | 动态，取决于集合大小         | 固定（通常 2-4 个）  |
| 代码形式   | `for x in options: dfs(...)` | `dfs(...); dfs(...)` |
| 典型题目   | #17, #46, #78                | #22, 树的 DFS        |

## 回溯 (Backtracking)

### 概念

回溯是递归在"枚举所有可能解"场景下的具体应用。它像一棵**决策树**：从根节点出发，每一层做一次选择，走到叶子节点时得到一个完整的解。

与普通递归的区别：回溯通常需要在递归返回后**撤销选择**（restore state），以便尝试其他分支。

> **剪枝 (Pruning)**：在递归中提前检查当前路径是否已经不合法，如果是，直接 `return` 而不继续深入。剪枝能大幅减少无效搜索，是回溯的关键优化。

### 通用框架

```python
result = []

def dfs(当前状态):
    # 终止条件：得到一个完整的解
    if 到达终点:
        result.append(当前路径)
        return

    for 每个可能的选择:
        # 剪枝：提前排除不合法的选择
        if 不合法:
            continue  # 或 return（取决于剪枝类型）

        # 做选择
        更新状态

        # 递归
        dfs(下一层状态)

        # 撤销选择（如果状态是可变对象，比如 list）
        还原状态

dfs(初始状态)
return result
```

> **注意**：如果状态是**不可变对象**（如 Python 的 `str`、`int`），每次递归传入新值，天然不需要撤销。如果状态是**可变对象**（如 `list`），需要在递归返回后手动撤销。

## 常用技巧总结

### 按层递推

用于"每层做一个独立选择，直到所有层都选完"的问题。

**代表题目**：17 - Letter Combinations of a Phone Number

**核心模式**：用 `index` 表示当前处理到第几层（第几个数字），每层遍历该位置所有可能的选项，递归到下一层。

```python
def dfs(index: int, path: str):
    # 终止：所有位置都已选完
    if index == len(digits):
        result.append(path)
        return

    # 枚举当前位置的所有字母
    for ch in phone[digits[index]]:
        dfs(index + 1, path + ch)  # str 不可变，天然撤销

dfs(0, "")
```

决策树示意（digits = "23"）：

```
              ""
         /    |    \
        a     b     c
      / | \ / | \ / | \
     d  e f d e f d e f
```

每条从根到叶的路径，就是一个合法的字母组合。

---

### 约束剪枝

用于"每步有多种选择，但需要实时检查合法性"的问题。

**代表题目**：22 - Generate Parentheses

**核心模式**：在每个递归分支入口处，先判断当前状态是否合法（剪枝），再继续递归；当状态完整时收集结果。

```python
def dfs(left_count: int, right_count: int, path: str):
    # 剪枝：括号数量超限，或右括号多于左括号（不合法前缀）
    if left_count > n or right_count > n or right_count > left_count:
        return

    # 终止：左右括号都用完，得到合法组合
    if left_count == n and right_count == n:
        result.append(path)
        return

    # 选择 1：加左括号
    dfs(left_count + 1, right_count, path + "(")

    # 选择 2：加右括号
    dfs(left_count, right_count + 1, path + ")")

dfs(0, 0, "")
```

剪枝去掉的分支（以 n=2 为例）：

```
")" → 右括号先于左括号，立刻剪枝
"(((" → 左括号超过 n，立刻剪枝
"())" → 右括号数量超过左括号，立刻剪枝
```

---

### 总结

| 技巧     | 核心机制                     | 状态是否需要撤销        | 典型题目 |
| -------- | ---------------------------- | ----------------------- | -------- |
| 按层递推 | index 逐层推进，枚举每层选项 | 否（str/int 不可变）    | 17       |
| 约束剪枝 | 实时检查合法性，提前截断分支 | 否（str/int 不可变）    | 22       |
| 显式撤销 | list 等可变对象需手动 pop    | 是（path.pop() 等操作） | 46, 78   |
