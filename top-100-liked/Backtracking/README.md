# Backtracking（回溯）解题思路总结

## 核心模板

回溯本质是 **DFS + 撤销**，所有题目都遵循同一个骨架：

```python
def dfs(参数):
    if 终止条件:
        result.append(path[:])  # 记录答案（注意拷贝）
        return

    for 候选选项 in 候选集:
        if 剪枝条件:          # 提前排除不合法分支
            continue
        path.append(选项)     # 做选择
        dfs(下一层参数)        # 递归
        path.pop()            # 撤销选择（回溯）
```

---

## 三种题型及对应套路

### 1. 子集 / 组合型（无序）

> 代表题：78. Subsets、39. Combination Sum

- 结果中元素**顺序无关**（`[1,2]` 和 `[2,1]` 算同一个）
- 用 `start` 下标控制"只往后选"，避免重复
- 每次递归传入 `start`，候选从 `start` 开始枚举

```
Subsets [1,2,3]:
            []
       /    |    \
     [1]   [2]   [3]
    /   \    \
 [1,2] [1,3] [2,3]
   |
[1,2,3]
```

| 题号                | 特点                                      | 细节                                        |
| ------------------- | ----------------------------------------- | ------------------------------------------- |
| 78. Subsets         | 每个元素只选一次，收集所有节点            | `dfs(index+1)`，每层都记录 path             |
| 39. Combination Sum | 元素可重复选，收集满足 sum==target 的叶子 | `dfs(index)`（不往后移），剩余值 < 0 时剪枝 |

---

### 2. 排列型（有序）

> 代表题：46. Permutations、17. Letter Combinations

- 结果中元素**顺序有关**（`[1,2]` 和 `[2,1]` 是不同答案）
- 每次从**全部候选**中选，但已用过的不能再用
- 用 `remaining` 列表或 `used[]` 数组跟踪哪些元素还可以选

```
Permutations [1,2,3]:
              []
         /    |    \
       [1]   [2]   [3]
      /  \   ...
  [1,2] [1,3]
    |
[1,2,3]
```

| 题号                    | 特点             | 细节                                                    |
| ----------------------- | ---------------- | ------------------------------------------------------- |
| 46. Permutations        | 数字全排列       | 每层从未用过的数字中选                                  |
| 17. Letter Combinations | 按位置映射字母表 | 每位数字对应一组字母，天然有序，直接 for 枚举该位的字母 |

---

### 3. 约束剪枝型

> 代表题：22. Generate Parentheses、131. Palindrome Partitioning

- 不是在固定候选集里选，而是**按规则构造**，违反约束立刻剪枝
- 剪枝逻辑是核心，写对了效率大幅提升

| 题号                         | 剪枝条件                    | 终止条件             |
| ---------------------------- | --------------------------- | -------------------- |
| 22. Generate Parentheses     | 左括号 > n；右括号 > 左括号 | path 长度 == 2n      |
| 131. Palindrome Partitioning | 当前子串不是回文 → 跳过     | index 到达字符串末尾 |

## 关键细节

### 为什么记录答案时必须写 `path[:]` 而不是 `path`

`path` 是列表，Python 里列表是**引用类型**。`result.append(path)` 只是把指向同一块内存的指针存进去，后续回溯继续修改 `path` 时，`result` 里已存的"答案"也会跟着变：

```python
path = []
result = []

path.append(1)
result.append(path)   # result = [[1]]，但存的是引用，不是副本

path.append(2)        # result = [[1, 2]]  ← 被改掉了！
path.pop()            # result = [[1]]     ← 又变回来了
```

`path[:]` 会创建一份**独立的新列表**，之后 `path` 怎么变都不影响已记录的答案。`list(path)` 效果相同。

| 要点         | 说明                                                |
| ------------ | --------------------------------------------------- |
| `path[:]`    | 记录答案时必须拷贝，否则后续回溯会改掉              |
| `start` 下标 | 组合/子集题防止重复，排列题不需要                   |
| 剪枝位置     | 越早越好，在 for 循环内、递归前判断                 |
| 选 vs 不选   | Subsets 的二叉结构：两条分支分别代表选/不选当前元素 |

---

## 快速判断用哪种套路

```
题目要求"所有组合/子集"（顺序无关）？
    → 子集/组合型，用 start 下标

题目要求"所有排列"（顺序有关）？
    → 排列型，用 remaining 或 used 数组

题目有明确的合法性约束（括号/回文）？
    → 约束剪枝型，优先想清楚剪枝条件
```
